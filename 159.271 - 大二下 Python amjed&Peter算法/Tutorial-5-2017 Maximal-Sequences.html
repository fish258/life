Subsequence Totals

If you have a vector (a list) of integers, one thing you might want to do is find which subsequence that, when summed, gives the greatest total.

For positive integers, this is easy - the more numbers there are the greater the total, so just add them all together: 

[ 3, 12, 5, 7, 2, 6]               # sum is 35

However if some of the numbers are negative, the problem is more complicated:

[ -3, 12, -7,  5, 9, -10, 2, 6]    # sum is 14

as you'll find by working with the vector above, you can't just stop when you hit a negative value as sometimes it is better to include it. The maximum subsequence total for the above list is 19.

Your task for this tutorial is to

  1. (1.5 marks) create an alorithm (a function) that finds the subsequence with the greatest total in a list of integers. The list is passed to the function as a parameter.
    e.g. 

    d = [ -3, 12, -7,  5, 9, -10, 2, 6]
print(maximal_subsequence(d))

  2. (0.5 mark) evaluate the big-O performance of your program. Is it O(n), O(log n), O(n2) or what?
    You can do this by either

    • Easiest: by inspection (looking at your code) and determining how the number of operations varies with n) as we've done in lectures
    • Simple: adding a counter to the inner-most loop, or
    • Little harder: by measuring the time (this will need you to look at the lectures slides)

    If you use the counter or measure the time in seconds, you'll need to figure out how the count or number of seconds relates to n (the length of the list).

Hints

  • To sum a list (or a part of one), use the sum() function that's provided by Python.

  • To sum part of a list (a subsequence), you can use slicing to extract part of a list and then sum that: 

    e.g.   x = sum( L[2:5] )  # x is the total of L[2]+L[3]+L[4]  (but not L[5])

Generating long test sequences

To evaluate the big-O performance using a counter or times, you'll need to generate long sequences. The following function does this by creating lists of integers with specified maximum values and length.

Test your function on short sequences, as it pretty well impossible to get the correct answer for long sequences by adding the numbers up yourself. For longer sequences, you'll have to trust your code :-) 

import random

def create_int_list(length, maximum_value):
    L = []
    for i in range(length):
       value = random.randint(1, maximum_value)

       # 30% of the time make the value negative and twice as large
       if random.random() < 0.33: 
          value = -value*2
       L.append(value)
    return L

print("Creating sequences of mostly positive integers")
list15   = create_int_list(15, 10)
list150  = create_int_list(150, 10)
# list5000 = create_int_list(5000, 15)

print("\n*********** 15 integers:\n",    list15)
print("\n*********** 150 integers:\n",   list150)
